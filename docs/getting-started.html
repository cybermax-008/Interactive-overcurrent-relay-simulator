Getting Started

The IDMT Overcurrent Relay Simulator lets you model inverse definite minimum time relay curves on a time-current characteristic (TCC) chart. You define a transformer, set up one or more relays on its primary or secondary side, and immediately see how each relay responds to fault current. The entire workflow runs in your browser with no server calls, no login, and no installation required.

This guide walks you through the five core steps: entering transformer data, configuring a relay, reading the log-log chart, coordinating multiple relays, and exporting your finished study. By the end you will have a working coordination study you can share or print.

Step 1 -- Set Transformer Parameters

Open the simulator and locate the Transformer panel at the top of the left-hand column. This panel contains four fields that define the power system source:

  • MVA -- The transformer rated apparent power in megavolt-amperes. This scales the available fault current. A larger transformer supplies more fault current at the same impedance.
  • Primary kV -- The high-voltage winding line-to-line voltage in kilovolts (e.g. 33 kV, 66 kV, 132 kV).
  • Secondary kV -- The low-voltage winding line-to-line voltage (e.g. 11 kV, 6.6 kV, 0.415 kV).
  • Z% -- The transformer impedance as a percentage on its own MVA base. Typical values range from 4% for small distribution transformers to 12% or more for larger units. A lower Z% means higher available fault current.

From these four values the simulator derives the three-phase symmetrical fault current on each side using the formula:

Ifault = MVA x 1000 / ( kV x sqrt(3) x Z% / 100 )

For the default 10 MVA, 33/11 kV, 8% impedance transformer this gives a primary fault level of approximately 2,187 A and a secondary fault level of approximately 6,561 A at 100% fault. You will see these values update instantly in the Results panel as you change any transformer field.

Tip: If the primary and secondary kV values do not form a valid ratio for the chosen transformer, a warning banner will appear. Double-check that your voltages correspond to the same transformer winding pair.

Step 2 -- Configure Your First Relay

Below the transformer panel you will find the Relay Cards. The simulator starts with four relays, but you can remove relays down to one or add up to eight using the Add Relay and remove buttons. Each relay card contains the following settings:

  • Label -- A free-text name displayed in the chart legend and results table. Use descriptive names like "Primary OC1" or "Feeder Backup" so your study is easy to read.
  • Side -- Select Primary or Secondary from the dropdown. This determines which voltage level (and therefore which fault current magnitude) the relay sees. Changing a relay's side recalculates its operating current immediately.
  • CT Primary -- The current transformer primary rating in amperes. This is the full-load current that produces 1 A (or 5 A) secondary on the CT. Typical values might be 200 A for a primary-side relay or 600 A for a secondary-side relay.
  • Pickup Multiplier -- The relay pickup current expressed as a multiple of the CT primary. The actual pickup (setting) current is I_set = CT_Primary x Pickup_Multiplier. A value of 0.50 on a 200 A CT means the relay picks up at 100 A. The minimum is 0.01.
  • TMS -- The Time Multiplier Setting controls how fast the relay trips along its chosen curve. Lower TMS values give faster tripping. The range is 0.05 to 1.0. Use the slider or type a value directly.
  • Curve Type -- Choose from seven IDMT curve types: four IEC 60255 curves (Standard Inverse, Very Inverse, Extremely Inverse, Long Time Inverse) and three IEEE C37.112 curves (Moderately Inverse, Very Inverse, Extremely Inverse). Each curve has different steepness and shape, affecting how the relay responds to different fault magnitudes.
  • Enabled -- The toggle switch at the top of each card. Disabled relays are hidden from the chart and results but their settings are preserved so you can re-enable them later.

Start with a single enabled relay. Set it to the secondary side, CT Primary 600 A, Pickup Multiplier 0.50, TMS 0.20, and IEC Standard Inverse. You should see a single curve appear on the chart immediately.

Step 3 -- Read the Chart

The main chart is a log-log time-current characteristic (TCC) diagram. The horizontal axis is fault current in amperes (log scale) and the vertical axis is operating time in seconds (log scale). Every enabled relay draws a curve showing its trip time across the range of possible fault currents.

Key elements on the chart:

  • Relay curves -- Each relay is drawn in a distinct color matching its card border. The curve starts just above the relay's pickup current and extends to the maximum plotted fault level. Steeper curves (Extremely Inverse) drop faster at high currents; flatter curves (Standard Inverse) maintain a more gradual slope.
  • Fault current line -- A vertical dashed line marks the current fault level as set by the Fault Level slider (5% to 100%). Where each relay curve intersects this line is that relay's operating time at the selected fault.
  • Operating points -- Colored markers on each curve at the fault current line show the exact trip time. These values are also displayed numerically in the Results panel.
  • CTI brackets -- When two or more relays are enabled, the chart draws time interval brackets between adjacent operating points. These show the coordination time interval (CTI) -- the margin between successive relay operations.

Hover your mouse (or tap on mobile) over any point on a curve to see a tooltip with the exact current and time values. The chart auto-scales its axes to fit all visible curves, but always maintains a sensible range so you can read the gridlines.

Below the chart, the Results Table shows trip times for every enabled relay at fault levels from 10% to 100% in 10% steps. This gives you a quick numeric overview without needing to read values off the chart manually.

Step 4 -- Coordinate Multiple Relays

Protection coordination means ensuring that the relay closest to a fault trips first, while backup relays trip with a deliberate time delay. The standard practice is to maintain a coordination time interval (CTI) of 0.3 to 0.4 seconds between successive relay pairs.

To set up a coordinated scheme:

  1. Enable two or more relays on the same or different sides. A typical arrangement has one or two relays on the secondary (closer to the load) and one or two on the primary (backup).
  2. Set the downstream relay (closest to the fault) with a lower TMS so it trips fastest. For example, a secondary relay at TMS 0.10.
  3. Set the upstream relay (backup) with a higher TMS so it trips after the downstream relay plus the CTI margin. For example, a primary relay at TMS 0.30.
  4. Adjust the Fault Level slider to sweep from low faults to maximum fault current. Watch the CTI brackets on the chart -- they should remain above your minimum acceptable margin (typically 0.3 s) across the full fault range.
  5. Check the CTI summary panel below the results. It reports the minimum CTI between each relay pair, highlighting any pairs that fall below the 0.3 s threshold.

If the CTI is too small at high fault currents, increase the TMS on the upstream relay. If it is too large, you may be able to reduce the upstream TMS for faster fault clearance. Changing the curve type also affects coordination: Very Inverse and Extremely Inverse curves provide better discrimination at high fault currents because their trip times drop more steeply.

Note: The simulator uses the IDMT operating time only. In a real protection scheme you must also account for circuit breaker operating time, CT errors, relay timing errors, and any overshoot time when determining your CTI margin.

Step 5 -- Export Your Study

Once you are satisfied with the coordination, the simulator offers several ways to save and share your work:

  • PDF Export -- Click the Export PDF button in the Report Settings section. The generated PDF includes a black-and-white version of the TCC chart (suitable for printing), the full results table, CTI analysis, and any report metadata you entered (company name, project title, document reference, remarks).
  • CSV Export -- Click Export CSV to download the trip time table as a comma-separated file. This is useful for importing into spreadsheets or other engineering tools for further analysis.
  • Share URL -- Click Share to generate a URL that encodes the entire simulator state (transformer data, all relay settings, fault level). Anyone who opens this link will see exactly the same configuration. This is the fastest way to send a study to a colleague.
  • Auto-Save -- Your settings are automatically saved to your browser's local storage. Close the tab and return later -- the simulator restores your last configuration.

Your settings are also automatically saved to your browser's local storage, so if you close the tab and return later, the simulator will restore your last configuration.
